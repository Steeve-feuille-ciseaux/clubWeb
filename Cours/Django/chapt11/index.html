🔁 Injection du fichier .sql

Dans le terminal de VS Code (après installation de sqlite3), tu peux faire :

sqlite3 db.sqlite3 < projectFC.sql

Cela exécute tout le script SQL dans la base.

👉 Le problème

En PowerShell, l’opérateur < (redirection d’entrée) n’est pas pris en charge de cette façon.

✅ 🔧 Solutions

Changer de terminal dans VS Code (recommandé)

Tu peux changer le terminal PowerShell pour utiliser cmd ou Git Bash, qui acceptent la redirection < comme tu essayais de faire.

Étapes :

  • Dans VS Code :
  • Clique sur le petit "+" dans la zone du terminal
  • Clique sur la flèche à droite du bouton +
  • Choisis "Select Default Profile"
  • Sélectionne :
    • Command Prompt (cmd)
    • ou Git Bash (si tu l’as installé avec Git)
  • Ferme et rouvre le terminal (Ctrl + `)
  • Puis tape ta commande :
sqlite3 db.sqlite3 < projectFC.sql

🔹 1. Vérifier si SQLite est installé

Dans le terminal VS Code (ou ton terminal système), tape :

sqlite3 --version

Si tu as un message comme "commande introuvable" ou "not recognized", alors il faut installer SQLite.

🔹 2. Installer SQLite selon ton système

🪟 Sous Windows :

  • Va sur : https://www.sqlite.org/download.html
  • Télécharge : sqlite-tools-win32-x86-*.zip
  • Dézippe le dossier dans un répertoire facile d’accès (par ex. C:\sqlite)
  • Ajoute le chemin à l’environnement :
    • Ouvre le Panneau de configuration → Système → Variables d’environnement
    • Dans PATH, ajoute le dossier où se trouve sqlite3.exe
  • Redémarre VS Code et essaie :
sqlite3 --version

🔹 3. Une fois SQLite installé

Place-toi dans le dossier où est ton fichier .sql, puis exécute :

sqlite3 db.sqlite3 < projectFC.sql

Cela va créer le fichier db.sqlite3 à partir du script SQL.

Ensuite (rappel rapide) :

  • Lancer Django avec ce fichier SQLite
  • Faire :
python manage.py inspectdb > monapp/models.py

Poursuivre avec vues, admin, etc.
