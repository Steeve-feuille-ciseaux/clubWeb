Configurer Git dans un projet Django

1. Initialiser Git

Place-toi à la racine du projet (là où se trouve manage.py) et exécute :

git init

2. Créer un fichier .gitignore

Ce fichier permet d’exclure les fichiers inutiles du suivi Git (comme les fichiers temporaires ou les migrations).

Voici un exemple adapté pour un projet Django :

# .gitignore
__pycache__/
*.pyc
*.pyo
*.pyd

env/
venv/
ENV/
*.sqlite3

*.log
*.pot
*.pyc
.DS_Store

# Django stuff:
*.db
*.sqlite3
/media
/staticfiles
/static_root
*.mo
*.pot

# VS Code
.vscode/

# Migrations
**/migrations/*.py
!**/migrations/__init__.py

3. Ajouter les fichiers

git add .

4. Faire un premier commit

git commit -m "Initial commit: Projet Django créé"

5. (Optionnel) Lier à GitHub

Crée un dépôt vide sur GitHub, puis :

git remote add origin https://github.com/ton_nom/projet-django.git
git push -u origin master

Bonnes pratiques

  • Fais des commits réguliers avec des messages clairs.
  • Ne versionne jamais de mots de passe ni de clés secrètes.
  • Utilise un fichier .env pour les variables sensibles (et ajoute-le au .gitignore).
