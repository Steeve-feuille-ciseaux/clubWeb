Comprendre un modèle Django : Article

Exemple de code

class Article(models.Model):
    titre = models.CharField(max_length=100)
    contenu = models.TextField()
    date_pub = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.titre

Explication des champs

  • titre : CharField – Texte court, max 100 caractères.
  • contenu : TextField – Texte long (corps de l’article).
  • date_pub : DateTimeField avec auto_now_add=True – Ajoute automatiquement la date/heure de création.

Autre types de champs Django courants

Type Django Utilisation Paramètres nécessaires / recommandés
CharField Texte court (nom, titre, email). max_length=... (obligatoire)
TextField Texte long sans limite spécifique (ex. : contenu d’article). Aucun requis, peut accepter blank, null
IntegerField Nombre entier. Peut inclure default, blank, null
FloatField Nombre décimal à virgule flottante. Peut inclure default, blank, null
BooleanField Valeur vraie ou fausse. Souvent avec default=True/False
DateField Date seule (ex. : date de naissance). Options utiles : auto_now, auto_now_add
DateTimeField Date et heure. auto_now, auto_now_add (non combinables)
EmailField Champ pour adresse e-mail (avec validation automatique). max_length recommandé
ImageField Téléversement d'image (nécessite Pillow). upload_to='chemin/' (obligatoire)
FileField Téléversement de fichier générique. upload_to='chemin/' (obligatoire)
ForeignKey Relation "un à plusieurs". to=Modèle, on_delete=models.CASCADE
ManyToManyField Relation "plusieurs à plusieurs". to=Modèle
