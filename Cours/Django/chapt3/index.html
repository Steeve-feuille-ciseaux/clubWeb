Django : base de données

1. Gérer une base de données

Créer un modèle dans pages/models.py :

Ouvre le fichier pages/models.py dans le dossier pages de ton application.

Remplace son contenu ou ajoute cette fonction à la fin du fichier :

            from django.db import models

            class Article(models.Model):
                titre = models.CharField(max_length=100)
                contenu = models.TextField()
                date_pub = models.DateTimeField(auto_now_add=True)

                def __str__(self):
                    return self.titre
détails de la commande

Créer la base de données :

                python manage.py makemigrations
                python manage.py migrate

Tu dois exécuter ces commandes dans le terminal, à la racine de ton projet Django, c’est-à-dire là où se trouve le fichier manage.py.

Ton arborescence de projet ressemble à quelque chose comme :


mon_projet/
│
├── manage.py
├── pages/
├── mon_projet/
├── db.sqlite3  <-- (si déjà créé)

Tu dois être dans le dossier mon_projet/ (le premier niveau), là où est manage.py.

Explications des commandes

1. Créer les fichiers de migration :

python manage.py makemigrations

Cette commande génère des fichiers dans le dossier migrations/ de chaque application. Ces fichiers décrivent les modifications à apporter à la base de données.

2. Appliquer les migrations à la base de données :

python manage.py migrate

Cette commande crée ou modifie les tables dans la base de données en fonction des migrations générées.

Astuce : Si le fichier db.sqlite3 (ou autre base de données) n'existe pas encore, il sera automatiquement créé lors de l'exécution de migrate.

Enregistrer le modèle dans l’admin :

Ouvre le fichier pages/admin.py dans le dossier pages de ton projet.

Ton arborescence de projet ressemble à quelque chose comme :


mon_projet/
├── mon_app/
│   ├── admin.py       ← C'est ici que tu ajoutera le code
│   ├── models.py      ← Ton modèle Article est ici
│   ├── views.py
│   └── ...
├── manage.py

Remplace son contenu ou ajoute cette fonction à la fin du fichier :

from django.contrib import admin
from .models import Article

admin.site.register(Article)

🧠 Explication ligne par ligne

Ligne Rôle
from django.contrib import admin Importe le module d'administration fourni par Django.
from .models import Article Importe le modèle Article défini dans ton fichier models.py.
admin.site.register(Article) Enregistre le modèle dans l’interface d’administration. Cela permet d’ajouter, modifier et supprimer des articles via /admin.
